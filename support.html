Quick troubleshooting

  • Nothing happens when I hold the hotkey. Open Settings → Hotkey and confirm a hotkey is bound and not shadowed by another app. The default is ⌃⌥ Space.
  • Sayz can't hear me. Check System Settings → Privacy & Security → Microphone and make sure Sayz is allowed. Then re-launch Sayz. You can also pick a specific input device in Settings → General.
  • The transcription appears in the Scratchpad but not in my other app. The Mac App Store build is intentionally scratchpad+clipboard only — paste it where you need with ⌘V. The direct-distribution build can additionally insert at the cursor; if it's installed, grant Accessibility access in System Settings → Privacy & Security → Accessibility.
  • Model download stuck or failed. Sayz verifies each model file against Hugging Face's published SHA-256s. If a file fails verification, the model is deleted and you can re-download from Settings → Model. Make sure you have at least 250 MB free for the Small model.
  • Transcription quality is poor. Try the Small model (M3/M4 recommended) instead of Tiny or Base. Add domain-specific terms in Settings → Vocabulary; the substitutions run before the text is inserted.

Reporting a bug

Diagnostic logs are stored locally on your Mac. To share them when reporting an issue:

  1. Open Sayz → Settings → Diagnostics.
  2. Click Copy Logs.
  3. Paste them into your email.

Logs never leave your Mac unless you explicitly copy and send them.
